# Dist Update SOP

## Purpose

Keep the checked-in `dist/` folder synchronized with the latest extension source so Edge can load the unpacked extension directly from `dist/`.

## Source of Truth

`dist/` is rebuilt from these paths in the repository root:

- `manifest.json`
- `src/`
- `assets/`
- `jsonl_ex.html`
- `jsonl_ex.js`
- `knowledge_base_tester.html`
- `knowledge_base_tester.js`
- `red_excel_generator.html`
- `red_excel_generator.js`

The rebuild script is:

```bash
node scripts/build-dist.mjs
```

That script clears `dist/` and copies the latest source files into it.

## When To Run

Run this SOP whenever any of these change:

- `manifest.json`
- anything under `src/`
- anything under `assets/`
- standalone tool pages in the repository root, including JSONL QA Editor, Knowledge Base QA Tester, and RED Excel Agent Flow

Also run it before committing if the change is meant to be reflected in the shipped extension folder.

## Update Steps

From the repo root:

```bash
node scripts/build-dist.mjs
```

Expected output:

```text
Built extension bundle in dist/
```

## Verification

Confirm `dist/` matches the latest source:

```bash
diff -qr src dist/src
diff -q manifest.json dist/manifest.json
diff -qr assets dist/assets
diff -q jsonl_ex.html dist/jsonl_ex.html
diff -q jsonl_ex.js dist/jsonl_ex.js
diff -q knowledge_base_tester.html dist/knowledge_base_tester.html
diff -q knowledge_base_tester.js dist/knowledge_base_tester.js
diff -q red_excel_generator.html dist/red_excel_generator.html
diff -q red_excel_generator.js dist/red_excel_generator.js
```

If these commands return no output, `dist/` is fully synchronized.

## Git Check

See what changed after the rebuild:

```bash
git status --short
git diff --stat -- dist
```

## Edge Reload

After updating `dist/`, refresh the unpacked extension in:

```text
edge://extensions
```

## Notes

- This project checks in `dist/` on purpose.
- `dist/` is not a separate compiled bundle pipeline here; it is a copied extension package generated by `scripts/build-dist.mjs`.
- If `dist/` looks outdated, rebuild first before debugging anything else.
